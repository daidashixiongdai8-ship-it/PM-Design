### 1. 需求背景

- **目标用户现状**：25-40 岁城市白领，睡前有刷手机习惯，导致入睡时间一再推迟。他们知道熬夜不好，但缺乏外部推力帮自己"断舍离"。现有方案（音乐歌单、竞品 APP）要么只提供声音却没有放下手机的引导，要么功能繁杂反而增加睡前操作负担。

- **核心痛点**：睡前无法主动放下手机，缺少一套从"打开 APP → 放下手机 → 安稳入睡 → 被轻唤醒 → 醒来获得正向反馈"的完整闭环。

- **当前问题汇总**：

| 问题点 | 问题描述 | 当前影响 |
|--------|----------|----------|
| 睡前手机依赖 | 躺下后惯性刷短视频/社交，不知不觉熬过入睡窗口期 | 入睡推迟 30-90 分钟，次日疲劳 |
| 缺乏放下手机的契机 | 现有白噪音 APP 只是播放声音，不解决"放下"这个动作 | 用户边听边继续刷手机，白噪音沦为背景音 |
| 入睡缺乏正向激励 | 早睡没有即时可见的回报，无持续动力 | 坚持几天后回到熬夜老路 |
| 选择过载 | 竞品声音库动辄上百种，睡前选半天 | 选声音的过程本身就在玩手机 |
| 起床体验粗暴 | 传统闹钟突然响起，无过渡 | 被惊醒后不适，破坏睡眠惯性 |
| 睡眠习惯无感知 | 用户不知道自己的睡眠是否规律 | 无法建立长期好习惯 |

---

### 2. 需求目标

- **业务目标**：
  - 用户从打开 APP 到放下手机的平均用时不超过 30 秒
  - 连续使用 7 天后，用户日均入睡时间较基线提前 20 分钟
  - MVP 上线 4 周内自然下载量验证核心价值假设

- **成功标准**：

| 指标 | 当前状态 | 目标状态 |
|------|----------|----------|
| 打开 APP → 启动入睡 转化率 | 未知 | ≥ 80% |
| 入睡启动 → 实际放下手机 完成率 | 未知 | ≥ 60% |
| 次日回访率 | 未知 | ≥ 40% |
| 7 日留存率 | 未知 | ≥ 25% |

### 3. 需求范围

- **包含**：
  1. 一键入睡引导（白噪音后台播放 + 屏幕渐暗 + 定时关闭 + 音频切换）
  2. 起床闹钟体系（设定入睡/起床时间 + 轻唤醒 + 闹钟 → 关闭闹钟弹出早安卡片）
  3. 全局音频卡片（三页面可见的极简迷你播放器 + 音频详情页）
  4. 音频库（白噪音含 Alpha 波 + 好梦故事 + 3 轨混音 + 自定义保存声音）
  5. 睡眠激励庄园（连续早睡奖励，白天在入睡页展示，醒后可查看）
  6. 早安卡片（金句 + 天气/下雨/带伞 + 穿衣建议 + 庄园奖励入口）
  7. 睡眠记录（入睡/起床时间、时长 + 日历视图 + 睡眠规律评分）
  8. 系统权限引导（自启动/省电白名单/锁定后台）

- **不包含**：
  - 账号登录与注册体系
  - 社区/社交功能（分享梦境、睡觉搭子等）
  - 鼾声录制与分析
  - AI 生成内容
  - 付费内容墙 / 会员订阅
  - 深度睡眠报告（心率和 HRV 等）
  - 宝宝卡/母婴服务
  - 冥想课程内容
  - 推送通知

### 功能 1：一键入睡引导（含昼夜切换）

**功能描述**

用户在【入睡】页操作。页面根据当前时段自动切换展示模式：夜间(20:00-6:00)展示极简入睡引导，白天(6:00-20:00)展示庄园与睡眠概览。

点击"开始入睡"后：屏幕渐暗(30s)，底部 Tab 隐藏，出现全局音频卡片 + 定时按钮 + 换音频按钮（均为纯图标）。用户可切换音频、设置定时。到达定时后音频淡出停止，记录入睡时间。播放中进入睡眠锁定状态，长按 3 秒退出。

**首次进入 — 系统权限引导：**

```
[首次点击"开始入睡"]
  ↓
[弹出系统权限引导弹窗]
  ├─ 内容：三个权限项说明
  │   - "应用自启动"：确保闹钟准时响起
  │   - "省电模式白名单"：防止后台被系统清理
  │   - "锁定后台任务"：保持音频持续播放
  ├─ 每项右侧：当前状态 + "去设置"按钮（跳转系统设置页）
  └─ 底部按钮："我知道了"关闭弹窗，进入入睡
```

**用户流程**

```
[夜间打开 APP → 入睡页]
  ↓
[显示：时间问候 + "开始入睡"按钮]
  ↓
[点击"开始入睡"]
```

### 功能 5：起床闹钟体系 + 早安卡片

**闹钟方案：** APP 前台运行生效（方案 B），通过系统权限（自启动/后台锁定）尽量保持前台活跃。

```
  ├─ 音量从 0 渐变至用户设定的轻唤醒音量（15 秒渐变）
  └─ 持续播放至 8:00
  ↓
[8:00 → 闹钟响起]
  ├─ 播放用户选择的闹钟铃声
  ├─ 音量为用户设定的闹钟音量
  └─ 持续播放，直到用户关闭
  ↓
[用户点击"关闭闹钟"]

[弹出早安卡片]
```

### 功能 6：睡眠激励庄园

**功能描述**

连续早睡达标 → 解锁庄园物品。奖励分为两类：每日小奖励（当天达标即永久获得一个小元素）+ 里程碑奖励（累计天数触发大件物品）。庄园在【入睡页-白天模式】和【我的页】两处展示，共享同一数据源。入睡页-夜间模式不展示庄园，我的页全时段可见。

**庄园可见条件（明确化）：**

| 场景 | 庄园画面是否可见 | 说明 |
|------|-----------------|------|
| 入睡页-夜间(20:00-6:00) | 不可见 | 仅显示极简入睡引导 |
| 入睡页-白天(6:00-20:00) | 完整可见 | 替代入睡按钮成为首页主角 |
| 我的页-任意时段 | 完整可见 | 位于早安卡片摘要下方 |
| 睡眠锁定屏 | 不可见 | 全黑屏 |
| 关闭闹钟后的早安卡片 | 不可见画面，仅文字入口 | 达标显示"庄园有新奖励，去看看 →" |
| 首次使用 无记录 | 可见但为空园 | 显示空地 + 树苗 + 引导文字 |

**激励规则：**

**每日奖励（当天达标即永久获得）：**

每天早睡达标，庄园中永久增加一个小元素。与里程碑奖励的区别是颗粒度——每日奖励让庄园"每天都丰富一点点"。

| 达标次数 | 每日奖励内容（随机一种） |
|----------|------------------------|
| 每次达标 | 小花 / 蘑菇 / 蝴蝶 / 萤火虫 / 小石子路一段 / 小鸟 / 四叶草 / 露珠 / 蒲公英 / 小石头 |

- 随机逻辑：从奖励池中随机抽取，避免短期重复（最近 7 次不重复）
- 每天最多获得 1 个每日奖励
- 每日奖励永久保留，不因中断而消失
- 已获得的每日奖励不可重复抽取（池子总量 30 个，够用一个月不重样）

**里程碑奖励（累计达标天数触发）：**

| 累计达标天数 | 解锁内容 | 类型 |
|-------------|----------|------|
| 1 天 | 一株小草 | 植物 |
| 3 天 | 一朵小花 | 植物 |
| 5 天 | 一棵小树 | 植物 |
| 7 天 | 白色栅栏 | 装饰 |
| 10 天 | 星星灯串 | 装饰 |
| 14 天 | 风车 | 装饰 |
| 21 天 | 小木屋 | 建筑 |
| 30 天 | 满园花开 | 终极 |

**达标判定：**
- 实际入睡时间（点击"开始入睡"的时间）≤ 用户设定的目标时间 ± 15 分钟缓冲
- 例：目标 23:00，22:45-23:15 之间点击开始入睡 = 达标
- 播放中途长按退出 = 不作数
- 中断：某天未达标 → 连续天数归零，已解锁物品不丢失

**用户流程**

```
[次日关闭闹钟 → 早安卡片显示达标]
  ↓
[达标 → 庄园自动添加 1 个每日奖励]
  ├─ 每日奖励 + 达到新里程碑 → 卡片显示"庄园有新奖励"(每日+里程碑)
  │    ↓
  │  [点击 → 跳转庄园区域 → 高亮光点闪烁(可能多个)]
  │    ↓
  │  [点击光点 → 弹出"获得了 [每日奖励名]！"或"恭喜解锁 [里程碑物品名]！"]
  │
  ├─ 每日奖励 但 未达新里程碑 → 卡片仅显示"昨晚早睡达标"，每日奖励静默添加
  │    ↓
  │  [用户去庄园查看 → 发现多了新元素(带微弱闪烁提示)]
  │
  └─ 未达标 → 卡片不出现，连续天数归零，庄园无变化
```

### 功能 7：睡眠记录与睡眠规律

**功能描述**

用户在【我的】页查看睡眠日历和睡眠规律评分。记录每日入睡/起床时间与时长。近一周数据分析睡眠规律。

**记录逻辑：**
- 入睡时间 = 用户点击"开始入睡"的时间
- 起床时间 = 第一次记录为闹钟关闭时间（轻唤醒开始 5 分钟后的闹钟关闭时间），用户可在日历中修正
- 睡眠时长 = 起床时间 - 入睡时间
- 修正：日历中每条记录可修正起床时间，入睡时间不可改

**睡眠规律评分：**

综合近一周（7 天）的入睡时间、起床时间、睡眠时长三个维度的稳定性：

```
评分 = 100% - (入睡时间方差贡献 + 起床时间方差贡献 + 时长方差贡献)

各维度贡献 = 该维度标准差 / 该维度均值 × 权重

权重：入睡时间 40% / 起床时间 30% / 时长 30%
```

逻辑示例：用户 7 天都在 23:00±10min 入睡、7:00±10min 起床、时长 7.5h±0.3h → 规律性 ≥ 90%

展示：
- 百分比环形图 + 大数字"85%"
- 下方一行小字"近一周睡眠规律"
- 数据不足 3 天时 → 显示"数据积累中，还需要 X 天"

**我的页展示布局：**

```
[睡眠规律]  环形图 85%
            近一周入睡/起床/时长综合评分

[统计卡片行]
┌──────────┬──────────┬──────────┐
│ 平均入睡  │ 平均时长  │ 连续早睡  │
│  23:15   │  7h12m   │   5 天   │
└──────────┴──────────┴──────────┘

[本周睡眠] ●●●○●●●  → 点击展开月历
         一二三 四 五六日
```

**用户流程**

```
[我的页 → 查看睡眠规律]
  ↓
[环形图展示百分比 + 简短评价]
  ├─ ≥ 90%："你的睡眠非常规律，保持下去！"
  ├─ 70-89%："睡眠还算规律，继续加油"
  ├─ 50-69%："睡眠不太规律，试试固定入睡时间"
  └─ < 50%："睡眠规律性偏低，从今晚开始调整吧"
  ↓
[点击日历入口 → 展开月历弹窗]
  ├─ 每天一个圆点(绿达标/灰未达标)
  ├─ 点击某天 → 详情(入睡/起床/时长/达标)
  └─ 点击修正起床时间 → 时间选择器 → 确认
```

**文案规范**

| 场景 | 文案内容 |
|------|----------|
| 数据不足(≤3天) | "数据积累中，还需要 X 天" |
| 规律 ≥ 90% | "你的睡眠非常规律，保持下去！" |
| 规律 70-89% | "睡眠还算规律，继续加油" |
| 规律 50-69% | "睡眠不太规律，试试固定入睡时间" |
| 规律 < 50% | "睡眠规律性偏低，从今晚开始调整吧" |
| 无记录 | "还没有睡眠记录，今晚开始你的第一次好睡眠" |
| 统计-平均入睡 | "平均入睡" |
| 统计-平均时长 | "平均时长" |
| 统计-连续早睡 | "连续早睡" |
